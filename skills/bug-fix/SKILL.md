# bug-fix — 검증 실패 자동 수정 루프

work 의 완료기준 검증이 실패하면 자동 호출된다. 에러코드 추출 → 자동 수정 → 재검증을 반복한다.

## 트리거
- work 의 완료기준 검증 실패 시 자동 호출 (또는 `/bug-fix`)

## 절차
1. **에러 추출**: 실패한 완료기준 명령의 stderr/stdout 에서 에러코드·메시지·파일:라인을 추출한다.
2. **자동 수정**: 근본 원인을 좁혀 코드를 수정한다. (임시 회피 금지 — 구조적 해결)
3. **재검증**: 실패한 완료기준 명령을 다시 실행한다.
   - 통과 → 다음 완료기준으로 / 모두 통과면 작업 완료.
   - 실패 → 시도 횟수 +1 후 1번으로.
4. **카운터**: 시도 횟수를 `.harness/run-{id}.md` 에 기록한다. **최대 5회** (hook 아님 — 스킬 내부 카운터, 설계 원칙 3).
5. **5회 초과 시**: 루프 **중단** → `/harness-check` **자동 트리거** ("이 실패가 하네스 문서의 잘못된 가이드 때문인가?" 진단 → Notion 기록). 코드 루프가 막히면 하네스 개선 루프로 넘어가 전체가 닫힌다.

## 원칙
- 검증·카운터는 hook 이 아니라 이 스킬 내부에서 처리한다.
- 상태 파일로 시도 횟수를 영속화해 세션이 끊겨도 카운트가 유지된다.
